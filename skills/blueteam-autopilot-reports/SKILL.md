# BlueTeam - Reports

Deterministic Markdown report generation from structured JSON data.

## When to Use

Invoke this skill when:
- Generating incident reports for UI display or ticket export
- Creating vulnerability prioritization summaries
- Proposing structured actions for human approval
- Producing runbook-aligned checklists

---

## Template Catalog

| Template | Purpose | Input Schema | Usage |
|----------|---------|--------------|-------|
| `incident-report.md` | Incident summary with attack chain | `schemas/incident-report.json` | `render-report.py --type incident` |
| `vuln-prioritization.md` | Vulnerability ranking by asset | `schemas/vulnerability-prioritization.json` | `render-report.py --type vuln` |
| `action-proposal.md` | Human approval proposal | `schemas/action-proposal.json` | `render-report.py --type action` |
| `runbook-checklist.md` | WAF triage checklist | `schemas/incident-report.json` | `render-report.py --type checklist` |

---

## Renderer Script

**Location:** `scripts/render-report.py`

**Purpose:** Deterministic Markdown renderer (replaces Dart StringBuffer operations)

**Usage:**
```bash
# Render incident report
python3 scripts/render-report.py --type incident --input report.json

# Render to file
python3 scripts/render-report.py --type incident --input report.json --output report.md

# Render vulnerability triage
python3 scripts/render-report.py --type vuln --input vulns.json

# Render action proposal
python3 scripts/render-report.py --type action --input proposal.json

# Render runbook checklist
python3 scripts/render-report.py --type checklist --input report.json
```

**Input Format:** JSON matching the schema in `schemas/` directory

**Output Format:** Formatted Markdown ready for display or ticket export

---

## JSON Schema Validation

All input data is validated against JSON Schema files before rendering:

| Schema File | Validates | Required Fields |
|-------------|-----------|-----------------|
| `schemas/incident-report.json` | Incident reports | `eventId`, `title`, `severity`, `aiSummary`, `rootCause`, `businessImpact` |
| `schemas/action-proposal.json` | Action proposals | `reasoning`, `recommendedPolicyId`, `expectedEffects`, `rollbackPlan`, `riskLevel` |
| `schemas/vulnerability-prioritization.json` | Vuln prioritization | `totalAnalyzed`, `rankedVulns`, `remediationSteps` |

**Validation Error Example:**
```
Error: 'eventId' is a required property
Failed validating 'required' in schema:
  - incident-report.json

Missing required fields: eventId
```

## Report Generation Workflow

### Option A: Agent-Driven Report (Recommended)

Use the `generate_incident_report` tool (Behavior 5b in the agent workflow):

1. **Call the tool**: `generate_incident_report(event_id, additional_context)`
   - Aggregates all investigation data in one call: event detail, alerts, assets, vulnerabilities, WAF, response policies, and compliance controls (NIST CSF + SOC 2)
   - Returns structured JSON with all sections needed for a full IR report

2. **Synthesize the report**: The agent uses the returned context to produce a comprehensive Markdown report with:
   - Summary, Attack Chain, Compliance Mapping
   - Blast Radius, Investigation Timeline, Confidence Rating
   - Recommended Actions (with policy IDs and risk levels)
   - Rollback Plan, Audit Trail

3. **Export**: Copy the Markdown output to a ticket system, compliance audit, or file.

### Option B: Template-Based Rendering

For deterministic rendering from pre-built JSON data:

### 1. Collect Data

From MCP tools or CLI scripts (for template-based rendering):
```bash
# Get event detail
aliyun sas describe-susp-event-detail --region "$REGION" --suspicious-event-id "$EVENT_ID" > event.json

# Or use MCP tool
get_security_event_detail(eventId="evt-xxx")
```

### 2. Transform to Schema Format

Map API response to report schema (for template-based rendering):
```python
# Example transformation
report_data = {
    "eventId": event["EventId"],
    "title": event["EventName"],
    "severity": event["Severity"],
    "aiSummary": "...",  # Generated by AI or extracted
    "rootCause": "...",
    "businessImpact": "...",
    "attackChain": [...],
    "affectedAssets": [...],
    "sourceIps": [...],
    "relatedCves": [...],
    "complianceControls": [...]
}
```

### 3. Render Report

```bash
python3 scripts/render-report.py --type incident --input report.json
```

### 4. Review and Export

- Review generated Markdown
- Export to ticket system or save to file
- Attach to incident record

## Example: Incident Report

**Input JSON:**
```json
{
  "eventId": "evt-20260614-001",
  "title": "SQL Injection Attack on Production WAF",
  "severity": "HIGH",
  "aiSummary": "Automated SQLi scanner detected targeting production web application.",
  "rootCause": "External attacker exploiting SQL injection vulnerability in search endpoint.",
  "businessImpact": "Potential unauthorized database access affecting customer data.",
  "attackChain": [

    {"stage": "Exploitation", "description": "SQLi payload detected: ' OR 1=1 --"},
    {"stage": "Impact", "description": "WAF blocked all malicious requests"}
  ],
  "affectedAssets": ["i-prod-web-01", "i-prod-web-02"],
  "sourceIps": ["203.0.113.42"],
```

**Command:**
```bash
python3 scripts/render-report.py --type incident --input example.json
```

**Output:** Formatted Markdown report with tables, sections, and compliance references

## Human Approval Workflow

Action proposals include mandatory approval sections:

1. Generate proposal JSON with `requiresApproval: true`
2. Render with `render-report.py --type action`
3. Review proposal (check for trusted network matches)
4. Obtain human approval signature
5. Execute policy with approval token

**CRITICAL:** Per SOC 2 CC6.8.3, NEVER execute response policies without explicit human approval.
